# 版本更新说明（2026-06-22）

## 本次完成项

- 新增 Profile 级“流量/到期来源”配置，可指定某个机场订阅 URL 作为最终 `/sub/:token` 响应头 `subscription-userinfo` 的来源。
- 修复本平台生成订阅时未向客户端透传流量和到期响应头的问题。
- 修复节点名称过滤未生效的问题：实际调用 subconverter 时改为传入 `exclude=` 参数，同时保留动态配置中的 `exclude_remarks=`。
- 在订阅源页和设置页提供清晰的“节点名称过滤”入口，支持填写 `官网|到期时间|剩余流量` 这类正则。
- 新增项目级 `AGENTS.md`，固化本仓库的验证、远程操作、GitHub 上传和 Release 规则。

## 接口、页面或模块影响

- `Profile` 新增 `subscriptionInfoSourceId` 字段；容器启动时会通过 `prisma/seed.ts` 自动为旧 SQLite 数据库补列。
- `PATCH /api/profiles/:id` 支持保存 `subscriptionInfoSourceId`，并校验该来源必须属于当前 Profile 且类型为机场订阅 URL。
- 删除订阅源时，如果该源被配置为流量/到期来源，会自动清空 Profile 上的引用。
- `/sub/:profileToken` 会根据指定来源或启用的机场订阅源生成 `subscription-userinfo` 响应头。
- `applyProfileNodeFilters` 会把 Profile 过滤正则传给 subconverter 的 `exclude` 参数。

## 上线后核对建议

- 登录 Web 后检查“订阅源”页右侧是否能看到“流量/到期来源”和“节点名称过滤”。
- 确认目标 Profile 已指定正确机场订阅源，或保持自动汇总。
- 使用 `curl -I` 检查 `/sub/:profileToken?target=clash` 是否返回 `subscription-userinfo`。
- 下载最终订阅 YAML，确认 `官网`、`到期时间`、`剩余流量`、`永久地址` 等说明节点已被过滤。
