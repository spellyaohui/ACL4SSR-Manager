# 版本更新说明（2026-07-01）

## 变更摘要

- 将 Docker Compose 内置 subconverter 镜像从官方 `tindy2013/subconverter` 替换为支持 AnyTLS 等协议的 `asdlokj1qpi23/subconverter:latest`。
- 修复 AnyTLS 等协议机场订阅无节点问题：当 subconverter `target=mixed` 转换结果为空时，`/nodes/:token` 会回退为直接拉取并解码原始订阅内容（含 Base64 节点包）。
- 进一步修复：`mixed` 返回不完整节点（如 AnyTLS 被旧版 subconverter 丢弃）时，也会优先使用原始订阅节点链接；仅对 Clash YAML 机场订阅继续走 `mixed` 剥离规则。
- 节点链接类订阅在最终 `/sub` 转换时改为直接把订阅 URL 传给 subconverter，避免 `/nodes` 纯文本回源被 subconverter 400 拒绝。
- 订阅源页面增加保存/启用/停用/删除/刷新元信息的即时提示与列表更新，无需手动刷新页面。

## 部署说明

已部署环境需重建 subconverter 与 rule-manager 容器：

```bash
docker compose pull
docker compose up -d --force-recreate subconverter rule-manager
```

生产环境若使用 Docker Hub 镜像，建议固定 tag（例如 `spellyaohui/acl4ssr-rule-manager:2026-07-01-fix3`），避免 `latest` 被镜像缓存延迟更新。

## 影响范围

- 使用 AnyTLS、Hysteria2 等新协议机场订阅的用户，生成订阅应能正常包含节点。
- 仍保留原有 `mixed` 转换路径，兼容传统 SS/VMess 等订阅；Clash 完整 YAML 机场订阅的行为不变。
